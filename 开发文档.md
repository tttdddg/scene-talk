# 最终交付目标

项目名称：

## SceneTalk——能看见的 AI 对话助手

一句话介绍：

> SceneTalk 通过摄像头理解用户当前看到的场景，听取用户的语音问题，并用文字和自然语音作出连续回答。

最终必须形成以下完整流程：

```text
开启摄像头
    ↓
用户点击语音按钮并提问
    ↓
浏览器识别用户语音
    ↓
自动截取当前摄像头关键帧
    ↓
前端压缩图片
    ↓
后端调用视觉模型
    ↓
返回简洁、准确的回答
    ↓
页面展示回答并语音播报
    ↓
用户继续追问
```

最终演示至少覆盖：

1. 摄像头实时预览；
2. 中文语音提问；
3. 自动截取当前画面；
4. 物品和颜色识别；
5. 场景描述；
6. 基于上下文的连续追问；
7. 文字回答；
8. 中文语音播报；
9. 图片压缩率和请求耗时展示；
10. 权限拒绝、请求失败等异常提示。



# 重要技术边界

## 2.1 DeepSeek-V4 的用途

DeepSeek-V4 只用于驱动 Claude Code：

- 生成项目代码；
- 修改代码；
- 执行测试；
- 检查构建错误；
- 帮助整理 README 和 PR 描述；
- 辅助代码审查。

不要默认使用 DeepSeek-V4 处理摄像头图片。

应用运行时需要另外调用支持图片输入的视觉模型。

## 2.2 视觉模型选择

优先方案：

```text
阿里云百炼 OpenAI 兼容接口
```

模型通过环境变量配置：

```env
VISION_API_KEY=your_api_key
VISION_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
VISION_MODEL=qwen3.5-plus
```

如果控制台中没有该模型，选择当前账号能够调用、明确支持图片输入的视觉模型，并只修改 `VISION_MODEL`。

代码中禁止写死模型名称。

## 2.3 视觉输入方式

不连续上传视频流。

使用：

```text
实时摄像头预览
+
每次用户提问结束后截取一张关键帧
```

这样既能满足“AI 看到摄像头中的内容”，又可以：

- 降低推理费用；
- 减少网络传输；
- 减少模型调用；
- 降低开发复杂度；
- 提高 Demo 稳定性。

设计文档中要主动解释这是端云协同和成本控制设计，不要把它描述成“来不及实现视频流”。

# 前端核心实现

## 9.1 摄像头

使用：

```typescript
navigator.mediaDevices.getUserMedia({
  video: {
    facingMode: 'user',
    width: { ideal: 1280 },
    height: { ideal: 720 }
  },
  audio: false
})
```

要求：

- 页面进入后不强制弹出权限；
- 用户点击“开启视觉助手”后申请权限；
- 权限成功后显示实时视频；
- 页面卸载时停止所有视频轨道；
- 摄像头未开启时不能发送视觉问题；
- 显示明确的摄像头状态。

状态：

```typescript
type CameraStatus =
  | 'idle'
  | 'requesting'
  | 'ready'
  | 'denied'
  | 'unavailable'
  | 'error'
```

## 9.2 图片截取

用户语音识别得到最终文本后：

1. 从 `<video>` 绘制到隐藏 `<canvas>`；
2. 限制最长边为 1024 像素；
3. 保持原始宽高比；
4. 转换为 JPEG；
5. 初始质量设为 0.72；
6. 如果仍超过 500 KB，再将质量降低至 0.6；
7. 记录原始估算大小和压缩后大小；
8. 只保留当前问题对应的一张图片。

禁止：

- 定时连续截图；
- 每秒上传图片；
- 保存用户摄像头图片到服务器；
- 将图片写入数据库或日志。

## 9.3 语音识别

使用浏览器：

```typescript
window.SpeechRecognition ||
window.webkitSpeechRecognition
```

配置：

```typescript
recognition.lang = 'zh-CN'
recognition.continuous = false
recognition.interimResults = true
```

状态：

```typescript
type SpeechStatus =
  | 'idle'
  | 'listening'
  | 'processing'
  | 'unsupported'
  | 'denied'
  | 'error'
```

行为：

- 用户点击麦克风按钮开始提问；
- 识别过程中显示临时文本；
- 得到最终文本后自动发起视觉问答；
- 识别结果为空时不调用模型；
- 处理中禁用重复点击；
- 请求结束后恢复可提问状态。

## 9.4 语音播报

使用：

```typescript
window.speechSynthesis
```

要求：

- 每次播放前调用 `cancel()`；
- 优先选择 `zh-CN` 中文语音；
- 语速设置为 `1.0` 到 `1.08`；
- 页面提供“停止播报”按钮；
- 用户开始新提问时停止旧回答；
- 播报失败不影响文字回答展示。

## 9.5 对话上下文

只保留最近 4 轮用户和 AI 对话。

前端 Store：

```typescript
interface ChatMessage {
  id: string
  role: 'user' | 'assistant'
  content: string
  createdAt: number
  status?: 'sending' | 'success' | 'error'
}

interface ConversationState {
  messages: ChatMessage[]
  currentTranscript: string
  lastSnapshot?: string
  isSubmitting: boolean
}
```

发送给模型时：

- 只发送文本历史；
- 不重复发送历史图片；
- 截断过长回答；
- 当前图片只对应当前提问。

# 视觉模型提示词

系统提示词：

```text
你是 SceneTalk，一个面向日常视觉辅助场景的中文视觉对话助手。

你的任务是结合当前图片、用户问题和少量历史对话，给出准确、简洁、自然的中文回答。

请严格遵守以下规则：

1. 只描述图片中能够合理确认的内容。
2. 如果无法确认，应明确说“我无法从当前画面中确定”，不得编造。
3. 不识别或猜测画面中真实人物的身份。
4. 如果用户使用“它”“刚才那个物品”等指代，可以结合最近对话理解。
5. 回答控制在 1 至 4 句话，适合直接进行语音播报。
6. 优先直接回答问题，不输出分析过程。
7. 对数量、颜色、文字等容易误判的信息，保持谨慎。
8. 如果画面过暗、模糊或主体不完整，应建议用户调整摄像头。
9. 不要使用 Markdown 表格。
10. 不要在回答前添加“根据图片”“从画面来看”等重复套话，除非确有必要。
```

视觉请求内容应包括：

```text
用户当前问题：
{question}

请结合当前摄像头关键帧回答。
```

不要要求模型返回长篇推理过程。

## 11.2 视觉风格

采用：

- 深蓝黑背景；
- 摄像头区域使用大圆角；
- 主色为明亮蓝紫色；
- 成功状态使用柔和绿色；
- 错误状态使用红色；
- 对话区域使用半透明卡片；
- 麦克风监听时显示呼吸动画；
- 请求模型时显示“正在观察并思考”；
- 不使用大量渐变和过度动画。

## 11.3 核心状态文案

空闲：

```text

```

```text

```

```text

```

模型处理中：

```text
SceneTalk 正在观察并思考……
```



# 指标面板

页面底部显示：

```text
压缩前：1.82 MB
压缩后：168 KB
压缩率：90.8%
视觉模型：2.36 s
端到端：3.08 s
上下文：2 / 4 轮
```

计算方式：

```typescript
compressionRatio =
  (1 - compressedBytes / originalBytes) * 100
```

端到端耗时从：

```text
获得最终语音文本
```

开始计时，到：

```text
收到完整模型回答
```

结束计时。

指标面板既是产品能力，也是视频中解释成本控制的关键内容。

# 异常处理

必须实现以下异常。

## 摄像头权限拒绝

提示：

```text
无法访问摄像头。请在浏览器地址栏中允许摄像头权限后重试。
```

## 麦克风权限拒绝

提示：

```text
无法使用麦克风。请允许麦克风权限，或检查系统输入设备。
```

## 浏览器不支持语音识别

提示：

```text
当前浏览器不支持语音识别，建议使用最新版 Chrome 或 Edge。
```

开发调试模式可以提供文字输入，但正式演示必须使用语音。

## 图片过大

前端再次压缩；如果仍过大，不发送请求，并提示重新调整画面。

## 模型超时

30 秒后取消请求：

```text
视觉分析等待时间过长，请重新提问。
```

## 模型限流

提示：

```text
当前请求较多，请稍等几秒后重试。
```

## 返回空内容

提示：

```text
这次没有得到有效回答，请重新描述问题。
```

## 重复点击

请求处理中禁用麦克风按钮，并显示处理状态。
